ScrumDo will release it's in January, 2011

We've seen too many open source projects out there post the source to their project, and then flounder. Often, it's because there isn't enough supporting material to make it a successful project.

Here at ScrumDo, we're taking the next few weeks to make sure that doesn't happen to us.

During this time we are:
  • Performing a security audit on the code (we are running a public version of this site after all...)
  • Writing our project goals / manifesto to help guide decisions
  • Creating a process to accept or reject feature ideas
  • Looking over the other open source software components we're using to make sure we're in license compliance
Follow Us On Twitter to be reminded.
